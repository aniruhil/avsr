Data Acquisition, Management, Analytics, and Visualization

Independent Study in Leadership & Public Affairs (MPA 6930.105)

What is this course about?

Think of this independent study as a hands-on workshop on "all things data" but with a special emphasis on (a) acquiring data from various sources, (b) "data munging" (a term coined by Josua Reich to describe the inescapable fact that roughly 80% of any data analyst's time is spent cleaning and organizing data in an analysis-friendly format), (c) data analysis, and of course, (d) conveying the resulting narrative by way of static and/or interactive visualizations.

What tools will we use?

We will use R and RStudio. You can download the latest daily build of RStudio for your operating system from here and the latest stable release of R from here.

Once we reach the stage where we are ready to build a coherent set of interactive graphics we will use Shiny. To deploy your app to the web you will need a free account. Sign up at and pick the "Free" plan. You can look at the various types of plans here, and if necessary we can pay $9 for one month (but I doubt we'll need to do so so).

Do I need to buy any books?

You could, if you wanted to, but my goal is to provide you with key readings and of course you'll get all the R code from this website so you shouldn't need any R-specific books. However, I have put together a list of some books (because the masters of data visualization recommend them very highly) and blogs, etc. I review every week.

Technology Setup

We use R and RStudio, and then cap things off with Shiny. Follow the sequence shown below to get started:

  1. Download the latest stable release of R for your operating system from here and install it on your machine. Acept the default settings though the installation process.
  2. Download and install the latest daily build of RStudio for your operating system from here.
  3. Setup a free account on shinyapps from here. Once your account is created, login to shinyapps. You will be greeted by a "Getting Started" page that will walk you through the R commands you need to run. Essentially, you need to execute the following code

    install.packages('devtools')
    devtools::install_github('rstudio/shinyapps')

    The next step is to authorize your account. The code on shinyapps will look something like that shown below and might be easiest to use the "Copy to clipboard" button to copy-and-paste the authorization code into R and then run the code.

    shinyapps::setAccountInfo(name='yourshinyappsaccountname',
         token='yourtoken',
         secret='')

    Now all you need is a shiny app to deploy and that is the end-goal of this course.
  4. You should have an OU Box account. If you don't, go ahead and activate your account from here. Then create a folder for this course. Make sure to save all files and data in this folder. If you want to avoid uploading/downloading your materials to the Box folder via a browser, go ahead and install the Box Sync app from here. If you do this you can just run everything from the Box folder on your machine without having to worry about saving your work every now and then.
  5. One more thing before we get started. We will be running and sharing everything via an RMarkdown file. To see how it works and to make sure it runs on your machine, go ahead and (a) download this file, and (b) open it in RStudio.
